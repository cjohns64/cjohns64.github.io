Lowpoly Skeleton

Lowpoly Skeleton Project Summary

For this project I sculpted a full human skeleton with the goal of getting “close enough” to the reference that it looks accurate. I definitely got some of the anatomy wrong, but the final sculpt looks convincing enough. The sculpt ended up with about 6 million triangles which made it unusable for most applications. I then went about retopologizing the skeleton and optimizing the mesh and I got the triangle count down to 14,356.

Tree Asset

Tree Asset Project Summary

For this project I wanted to make a convincing CG tree that I could use in other projects. I had a lot of problems managing the complexity of the tree, since it is too computationally expensive and too time consuming to model each leaf. I ended up modeling a single branch, and then instancing that branch several times to form the tree. The instances were placed by hand.

Wheels of the Primal Gates

Wheels of the Primal Gates Project Summary

In this project, I made my interpretation of a location in one of my favorite games. In the game, Conquest of Elysium 4 by Illwinter Game Design, it is described that the primal gates (the portals that connect the elemental planes) are guarded by the wheels of Storm and Stone and the wheels of Frost and Flames. I had the most difficulty getting the wheel of Storm and Stone to stand out against the background, as well as how to depict elemental frost in a way that was instantly recognizable. One of the biggest successes of this project was that I figured out how to depict fire and clouds without resorting to simulations.

Bone Dragon

Bone Dragon Project Summary

This was my second attempt to make a dragon. The reasoning I had was if I sculpted it from the ground up (bones, then muscles, then scales) it would achieve a more realistic result. I didn’t anticipate that the final form would not match what I was going for. The project was a big success anyway, since I learned a lot about anatomy, sculpting, and retopology.

Solaren Video Remake

Solaren Video Remake Project Summery

For this project I made a completely new set of 3D animations for the Solaren corporation. The animation was based off of an earlier animation for the structure and most of the narration script. The new animation, however, does not contain any assets from the original animation. The animation contains 7 distinct scenes, with 3 distinct environments. All of the space scenes were done with Blender’s real-time renderer Eevee, while the SLS launch scene and the ground station scene were done with the Cycles renderer. The ground station scene is particularly interesting since its large scale required me to develop techniques for managing large scenes.

Frigate Project

Frigate Project Summary

I have always been fascinated by fully-rigged ships and for this project I made one in Blender. I learned about the rigging and layout of fully-rigged ships by reading Wikipedia articles. I didn’t notice the hull depth I was modeling was only to the water line of the reference ship until I was done with the rigging. I hadn’t setup the project organization correctly in order to support the large scale modifications needed to fix it. I learned a lot about how to use the non-destructive modeling workflow with this project.

Dragon Project 01

Dragon Project 01 - Project Summary

This is the first Blender project I worked on after learning the basics of the software. I didn’t know about Blender’s sculptor or texture painter while working on this project, so I modeled the dragon by moving vertexes around by hand and I textured it by UV unwrapping it and trying to paint in the bounds of the unwrap. I had previously learned 3D modeling using Google Sketch Up, so I thought the only way to model was drawing vertexes, edges, and faces. This probably slowed down my development in the short term, but I have found my familiarity with this method is very helpful for retopology.
